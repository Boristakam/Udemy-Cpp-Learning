c++ strings
they are objects!!!!
need string header file 
included in standard namespace 
they're dynamic and can change in size unlike c type strings 
work with input and output strings 
safer .. provide bound checks like vectors

declare with type 'string' and not char 
they are always initialise whether the user does so or not (means the variable will never contain random value if user doesnt init) 
can copy string into another without using the function like in c style. i.e. string s1 {s2} where s2 is another string 

string s1;              //empty
string s2 {"Frank"}     //Frank
string s3 {s2};         //Frank
string s4 {"Frank", 3}; //Fra
string s5 {s3, 0, 2};   //Fr     -> from first characher of s3 assign 2 characters to s5
string s6 (3, 'x');     //xxx    -> constructor style initialisation with the round brakets intead of the curlies 

use the + operator to concatonate c++ strings 
Note you can only concatonate the c++ string variables not the strings themself (a max of 1 string (not var name) is allowed in the concatenation operation)
     because the rule says you cannot concatonate more than one c style strings use c++ concatenation method. same applies for comparison operators 

similarly to vectors and array you can use the .at() or subscript operator [] to access string elements. at() method is bound 
can also use 'range base for loops' on them and a lot more...

.substr() use to extract parts of a string  i.e. cout << str_1.substr(0, 4);  this functions takes the 2 param, start index and length
     start index is zero
     cout << s2.substr (1, 3);  // "ran"

.find() takes a string parameter and return the interger representing the starting position of a char or substring within a 
    cout << s2.find ("ra"); // 1 -> meaning the searched item/word start at element idex 1 of the string that is being searched 
    cout << s2.find ("F", 3) // string::npos  -> meaning no position available for "F" starting form index 3

.rfind() searches from the end of the string and works as .find() above 
.erase() takes starting index and length 
.clear() deletes all character in the string leaving it empty 

.length() returns size of the string. Note .size() is for vectors and arrays not strings, but sizeof() is for all and return size in bytes
