they determine when and where an identifier can be used 
used static and lexical scoping (means scope is determined the same way you read a program)

local or block scope:
    identifiers declared in a block {}
    function param have block scope
    only visible within the block {} where declared 
    function local variables are only active while the functioin is executing 
    local variables are not preserved between function calls 
    with nested blocks, inner blocks can see out but outer blocks cant see in!!!!!!

    exception to static local variables declared with the 'static' qualifier, 
        value is PRESERVED between function calls and only init the first time function is called
        there automatically set to 0 if no initialisation is provided  


Global scopes:
    visible to all parts of the program after global variable has been declared 
    global constant are much better practice than global variables (because changing it will affect many functions!!!!)
