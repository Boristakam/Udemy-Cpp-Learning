functions use an area in memory called the 'function call stack' or 'program stack' 
    Stack is analagous to a stack of books 
    therefore uses the LIFO system (last in first out).. meaning you must revove a book(s) before accessing the ones below it 
    It uses the terms push and pop to add and remove and item from the top of the stack respectively 

for c++ program those items are called stack frames or activation record (collection of information that represent an active func (parameters, return address, etc..))
    functions must return control to the function that called them 
    each time a function is called we create an new activation record and push it on stack
    when a function terminates we pop the activatioin record and return 
    local variables and function parameters are allocated on the stack 



Note: 1) the call stack is finite in size so if you activate too many functions on the call stack it's possible to run out of stack space.. and the program terminates!!
      2) function call overhead is when the execution time of a function is less than the switching time from the function call of the caller 
            avoided by telling the compiler to use inline code (it generates inline assembly code.. much faster.. but could cause bloat as code is being duplicated)
            but compilers are so sophisticated now that will probably inline without any suggestion 

            inline int add_numbers(int a, int b) //function definition 
            {
                int return a + b;
            }

            inline functions are usually defined in headers (.h) since they should be access by any file that needs them
