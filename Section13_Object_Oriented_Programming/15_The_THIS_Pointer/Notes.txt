stepping through the code in the debugger weve noticed an identifier call 'this' in the window pane
'this' is a reserved keyword
contains the addr of the obj - so it's a pointer to the obj being worked on
it can only be used in the class scope 
all member access is done via the 'this' pointer
can be used by the progrmmer to: 
    access data member and methods
    determine if two objs are the same (more in the next section of the course)
    yield the current obj being worked on by dereferencing it (*this)

                        // so instead of    balance = balance   we would neet to write  this->balance = balance
    }
