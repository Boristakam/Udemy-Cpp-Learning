constructors
    special member methods invoked during object creation 
    useful for initialisation of private attributes
    it's a public method with same name as class 
    no return type is specified 
    can be overloaded (can have multiple constructors of same name differentiated by their parameter/passed arguments)
    i.e.    Player();
            Player(string z, int b); // the parameter could be private attributes of the class, therefore calling this constructor allows to initialise them all at once, 
                                     // rather than using the dot method or arrow method to access public attributes or calling public method that have code to access and change private members 
            Player(int x);

Destructors 
    also special public member methods that have the same name as the class proceeded with a tilde (~)    i.e ~Player();
    invoked automatically when an object is destroyed (i.e a pointer object to a memory addr on the heap) --- might be related to the delete keyword used previouly to release memory on the heap
    no return type as the constructors
    no parameters unlike the constructors
    only one destructor is allowed per class (cannot be overloaded), C++ wouldn't know which to call
    useful to release memory, close files, and other resources
