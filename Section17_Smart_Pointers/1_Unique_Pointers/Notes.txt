Simple smart pointer that is very efficient
std::unique_ptr<int> p1 // means that the pointer p1 will manage an integer on the heap 
unique_ptr<T>
    points to object of type T on the heap
    is unique - there can only be 1 uniqur_ptr(T) pointing to the object on the heap 
    owns what it points to 
    cannot be assigned or copied (so copy constructor and assignment operators are not available)
    can be moved (great for vectors)
    when the pointer is destroyed, what it points to is automatically destroyed

has many methods:
     the .get() method returns the address of the managed object 
     the .reset() method set the pointer to null and the momory is points to is released 
     you can check if the unique pointer is initialised 
        if (ptr)
        { 
            //statements 
        }

you can create vectors of unique pointers but you cant simply use to push_back method() because it copies content 
    you have to use the .move() method in combination vec.push_back(std::move(ptr)) 
    ptr has transfered ownership of the pointer to vec and ptr is set to null pointer

can use the make_unique<data_type>(data) to initialise smart poiters 
it replaces the use of the "new" keyword
